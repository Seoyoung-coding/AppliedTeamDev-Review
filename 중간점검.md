## 초안과 수정후 변경사항
Controller → DTO → Service → Domain(Entity) → Repository 순서로 개발한다는 점

## Content
### domain - Controller
#### 바뀐점 1
```
private String title (content, price, discountRate등)
```
필드는 null 이 될수 없기 때문에
```
@Column(nullable = false)
private String title;
```
즉, DB 레벨에서 NULL을 허용하지 않는다

### 바뀐점 2
```
private LocalDateTime createdAt
```
둘의 차이점은 뭘까?
```
private LocalDateTime createdAt =  LocalDateTime.now();
```
엔티티 생성 시점에 무조건 생성 시간이 필요하다면 위와 같이 쓰는게 맞다
1번의 경우 기본값이 null이 될 수 있음
Java 레벨에서 null 방지

### 바뀐점 3
```
@Builder.Default
private Integer viewCount = 0;
```
Lombok @Builder는 필드 초기화 코드를 무시하고 builder에 직접 전달된 값만 세팅함 (전달 안 하면 null)
즉, 값에 null이 없게 하고 디폴트값을 0 부터 시작하겠다고 설정함

### 바뀐점 4
Java <-> JPA <-> DB
```
@Enumerated(EnumType.STRING)  //Enum을 문자열로 DB에 저장
@Column(name = "status")
private ContentStatus contentStatus = ContentStatus.INACTIVE; // 컨텐츠 생성시 기본이 inactive 상태
```
Content 엔티티의 contentStatus 필드는 Java에서는 enum, DB에서는 문자열 컬럼, 기본값은 INACTIVE로 시작하는 구조가 된다
> 외부 노출 위험이 있어서 (?) => 좀더 알아보자
